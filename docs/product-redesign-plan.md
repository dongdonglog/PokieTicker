# 产品改造计划

## 目标
把当前项目从“面向固定数据源的美股新闻分析工具”改造成“可导入任意市场走势数据的 AI 图表分析工作台”。

默认体验继续保留美股 `Polygon` 路线，但产品核心不再依赖单一市场或单一在线数据源。

## 为什么要改

当前问题已经很清楚：

- 美股链路相对完整，因为 `Polygon` 同时提供搜索、行情和新闻
- A 股链路不稳定，在线数据源容易遇到权限、限流、反爬或断连
- 前端看板本身没有问题，真正限制产品扩展的是数据输入方式

因此，应该把“看板”和“数据来源”解耦。

## 新定位

PokieTicker 未来应具备两种模式：

1. `默认模式`
   使用现有美股数据和新闻链路，维持当前演示体验

2. `导入模式`
   用户自行导入任意市场的 OHLC 数据，由前端统一展示，AI 负责解释和问答

## 设计原则

- 前端看板尽量保持市场无关
- 后端 API 统一读本地 SQLite
- 在线数据源只负责“灌库”，不直接决定产品可用性
- AI 层可以在“有新闻”和“无新闻”两种情况下工作

## 核心改造方向

### 1. 数据源抽象

已开始引入 `backend/market_data.py`，后续统一通过 provider 接口管理：

- `polygon`
- `ashare`
- `imported`

目标是让 `stocks.py`、`pipeline.py`、`bulk_fetch.py`、`weekly_update.py` 都不直接依赖某一个市场实现。

### 2. 通用走势导入

新增本地导入能力，允许用户导入任意市场的 CSV/JSON 数据。

建议最小字段：

- `symbol`
- `date`
- `open`
- `high`
- `low`
- `close`
- `volume`

建议新增：

- `backend/import_ohlc.py`
- `docs/import-format.md`
- `data/imports/` 示例文件

### 3. 市场无关的前端展示

前端继续复用现有工作区：

- 主图表
- 新闻/信号/预测侧栏
- 区间分析
- 相似走势

但股票选择器不再假设固定市场分组。对于导入模式，优先展示：

- 本地已导入 symbols
- 名称 + 代码
- 可选市场标签

### 4. AI 提问替代部分固定交互

对于没有新闻的数据，AI 仍可回答：

- “这段为什么跌？”
- “这段走势像不像洗盘？”
- “最近波动的关键特征是什么？”

输入可以来自：

- OHLC 区间统计
- 技术指标
- 用户备注或自定义事件
- 若存在新闻，则附加新闻上下文

## 分阶段实施

### Phase 1

- 默认恢复美股体验
- 保留现有 `Polygon` 工作流
- 稳定当前看板和新闻显示

### Phase 2

- 完成通用 OHLC 导入脚本
- 定义 CSV/JSON 导入格式
- 让导入数据可直接出图

### Phase 3

- 前端增加“导入数据模式”
- 股票选择器支持本地导入 symbols
- 不再依赖固定市场分组

### Phase 4

- 加入 AI 提问入口
- 无新闻时基于走势和技术特征解释区间
- 有新闻时继续叠加新闻上下文

### Phase 5

- 再决定是否继续补 A 股新闻源
- 再决定是否保留 A 股在线抓取链路
